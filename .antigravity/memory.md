## What this project is

A **fork of Chatwoot** (open-source customer support platform) with a custom **ElevenLabs Voice Assistant** integrated into the live-chat widget. When a user opens the widget and the text input is empty, a microphone button appears that lets them talk to an ElevenLabs AI agent.

- **Upstream:** Chatwoot (Rails backend + Vue widget)
- **Custom layer:** Voice button + agent integration on top of widget
- **Deployment:** Docker (`docker compose up -d`), dashboard at `http://localhost:3000`

## Key configuration

ElevenLabs agent ID has **two layers** — runtime env var wins, otherwise hardcoded fallback:

```js
// custom/widget/store/modules/appConfig.js:36
agentId: import.meta.env.VITE_ELEVENLABS_AGENT_ID || 'agent_6601kc1fqeecfc88s7d52jde0syq'
```

Build args injected via `build.sh`:
- `VITE_ELEVENLABS_AGENT_ID`
- `VITE_ELEVENLABS_VOICE_ID`
- `VITE_ELEVENLABS_AGENT_NAME`

Changing agent ID requires a **rebuild** (Vite inlines env at build time, not runtime).

## How to build / deploy

```bash
./build.sh                                     # default agent
./build.sh agent_xyz voice_abc "My Agent"      # custom args
docker compose up -d                            # bring up stack
```
